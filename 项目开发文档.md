# 通用 AI 角色扮演轻问答网页 —— 项目需求文档

## 1. 项目概述
### 1.1 项目名称
**RoleChat（或自定义命名）** —— 通用 AI 角色扮演对话平台

### 1.2 项目目标
开发一个**通用型 AI 问答网页**，支持用户通过**设定世界观与角色**进入沉浸式角色扮演对话。前端界面需嵌入一套自行设计的**功能图标（12-15 个）**，以完成 UI 设计实验要求。后端调用 DeepSeek API（或其他兼容 API），实现智能回复，并保证图标风格统一、交互状态完整。

### 1.3 核心特色
- **角色扮演引擎**：新建对话时必须填写世界观、AI 角色设定、用户角色设定，系统将自动拼接为系统提示词。
- **通用 API 接口**：前端不绑定特定 AI 品牌，仅通过标准 Chat Completion 接口调用，可轻松切换后端模型。
- **沉浸式 UI**：对话界面模拟聊天应用，切换角色时伴随视觉变化（如头像、气泡颜色）。
- **图标设计实验载体**：网页所有功能入口均由自行设计的矢量图标承载，满足 12-15 个图标、多状态设计、可识别性测试等教学要求。

## 2. 功能模块与页面结构
### 2.1 全局底部导航栏（4 项）
| 图标名称     | 功能描述                 |
| ------------ | ------------------------ |
| 对话         | 进入对话历史列表         |
| 角色         | 管理预设角色             |
| 发现         | 浏览公共世界观/角色模板  |
| 我的         | 个人设置与数据           |

### 2.2 对话列表页
- 展示所有历史对话卡片（标题自动截取世界观首句）
- 支持操作：**新建对话**、**搜索对话**、**删除**、**置顶/收藏**
- 空状态时显示引导文案

### 2.3 对话界面（核心页面）
- **顶部栏**：当前世界观名称、角色切换按钮
- **消息气泡**：区分用户与 AI，不同角色可设置不同气泡颜色/头像
- **输入区**：文本输入框 + **发送按钮**（主要交互）
- **消息操作**（长按或悬停气泡出现）：**重新生成**、**复制**、**点赞/踩**
- **角色切换面板**：点击顶部角色图标可临时切换 AI 说话风格

### 2.4 角色管理页
- 查看已保存的角色卡（名称、世界观、性格标签）
- 操作图标：**编辑**、**删除**、**分享**、**设为默认**
- **新建角色**入口（跳转至角色创建流程）

### 2.5 角色创建/编辑页
- **世界观设定**输入框（图标：书）
- **AI 角色名称、性格、口癖**设定区（图标：用户肖像+齿轮）
- **用户扮演身份**输入框（图标：面具）
- **预览对话**（可选，图标：眼睛）
- **保存角色**（图标：书签）

## 4. 视觉设计与图标规范
### 4.1 整体风格
- 采用**极简线性风格**，图标以描边为主，必要时可部分填充。
- 主题色：`#1A73E8`（主蓝），辅色：`#F5F5F7`（背景）、`#8E8E93`（默认图标）、`#D1D1D6`（禁用态）
- 字体：系统默认无衬线字体（如 PingFang SC、Inter）

### 4.2 图标设计规范
- **尺寸**：基础画布 `24×24px`（移动端 @1x），实际导出 `48×48px` 用于高清屏。
- **线条粗细**：统一 `1.5pt`（2px 在 48px 画布下）
- **圆角**：外圆角 `2px`，内部细节圆角 `1px`
- **正负形平衡**：图标重心居中，视觉面积相近
- **多状态设计**：
  - 默认态：描边 `#8E8E93`
  - 点击态/悬停态：填充 `#1A73E8`（或深色背景上反白）
  - 禁用态：描边 `#D1D1D6`，透明度降低
- **导出格式**：SVG（矢量）、PNG（1x/2x/3x），附带颜色标注

## 5. 技术架构与开发流程
### 5.1 技术选型
- **前端框架**：纯 HTML/CSS/JS 或轻量框架如 Vue/React（便于部署和维护）
- **图标渲染**：直接使用 SVG 内联或 `<img>`，通过 CSS 类切换状态
- **API 调用**：前端通过 `fetch` 调用 DeepSeek Chat Completion API（或其他兼容接口），需做简单后端代理或使用支持前端调用的边缘函数（如 Cloudflare Workers, Vercel Functions）保护 API Key
- **部署平台**：Vercel / Netlify / GitHub Pages（静态站）+ 后端代理服务

### 5.2 开发步骤
1. **需求确认与图标设计**（已完成）
2. **搭建项目脚手架**（选择框架，初始化仓库）
3. **页面布局开发**（移动端优先，响应式适配）
4. **实现核心交互**：
   - 新建对话时弹出角色设定表单
   - 拼接 system prompt 调用 API，流式输出回复
   - 对话存储于本地（localStorage）或简易后端
   - 角色切换、消息操作等功能
5. **集成图标**：将所有设计好的 SVG 图标置于组件中，实现多状态切换
6. **联调测试**：测试 API 连通性、图标点击态、禁用态表现
7. **部署上线**：配置代理服务，部署前端至公开 URL
8. **用户测试与优化**：邀请同学进行图标语义理解测试，记录误读率并调整设计

### 5.3 API 集成关键点
- 请求地址：`https://api.deepseek.com/v1/chat/completions`（或自行代理的 URL）
- 请求头：`Authorization: Bearer YOUR_API_KEY`
- 请求体示例：
  ```json
  {
    "model": "deepseek-chat",
    "messages": [
      {"role": "system", "content": "你将扮演一位中世纪管家..."},
      {"role": "user", "content": "你好，今天天气如何？"}
    ],
    "stream": true
  }
  ```
- 流式输出处理：解析 SSE（Server-Sent Events），逐字显示到气泡中

## 6. 部署与访问要求
- 最终成果需部署为一个**可公开访问的 URL**，方便教师和同学浏览及测试。
- 推荐部署方案：
  - 前端静态资源托管在 Vercel/Netlify，自定义域名或使用其分配的子域名。
  - API 代理使用 Vercel Serverless Function 或 Cloudflare Workers，隐藏 API Key。
- 提交时需提供：**网页链接**、**设计源文件（Figma/Illustrator 等）**、**实验报告**（按指导书要求含图标设计稿与标注）。

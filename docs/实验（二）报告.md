# 实验二 产品图标设计与制作 —— 实验报告

> **项目名称**：RoleChat —— 通用 AI 角色扮演轻问答网页  
> **设计工具**：草图手绘 + SVG 代码实现  
> **实验日期**：2026 年 5 月

---

## 一、设计需求

RoleChat 是一款 AI 角色扮演对话平台。需要一枚品牌产品图标（App Icon），用于网页 favicon、PWA 安装图标、GitHub Pages 展示等场景。图标需在 16×16 到 1024×1024 像素范围内保持可识别性，并传达「AI」+「对话」+「角色扮演」三大产品基因。

---

## 二、设计定位

### 2.1 品牌分析报告

#### 2.1.1 品牌关键词法

| 层级 | 关键词 | 提取逻辑 |
|------|--------|----------|
| 核心功能 | AI 对话、角色扮演 | 产品最核心的两个动作 |
| 情感感知 | 沉浸、陪伴、友好 | 用户使用时的情绪体验 |
| 视觉联想 | 聊天气泡、面具、字母 R | 直接对应的视觉符号 |

核心关键词收敛为 3 个：**对话 · 角色 · AI**

#### 2.1.2 竞品对比

| 竞品 | 图标类型 | 优点 | 对 RoleChat 的启示 |
|------|----------|------|-------------------|
| ChatGPT | 中心对称星形 | 简洁、科技感强 | 避免纯科技冷感 |
| Character.AI | 彩色人物面孔 | 角色感强、活泼 | 可以突出角色的亲和力 |
| 微信 | 绿色对话气泡 | 识别度极高 | 对话气泡是最直观的聊天隐喻 |
| Slack | 四色井字 | 品牌色醒目 | 品牌色一致性很重要 |

**差异化定位**：不做纯科技工具类图标（如 ChatGPT 的冷感星形），也不做纯社交（微信的单一气泡），而是融合「对话（气泡）」和「角色（面孔/字形）」两个维度。

#### 2.1.3 用户画像

| 属性 | 描述 |
|------|------|
| 年龄 | 18-28 岁 |
| 兴趣 | 二次元、游戏、同人创作、AI 工具 |
| 场景 | 课余/通勤时打开，进行沉浸式角色扮演对话 |
| 审美偏好 | 现代简洁、有品牌辨识度、不要过于卡通 |

#### 2.1.4 情绪板（Mood Board）

情绪板收集关键词：**蓝色渐变**（科技+信任）、**粗圆线条**（友好、亲和）、**单一白色图形**（iOS 风格）、**微投影**（层次感）。参考品牌：Stripe、Notion、iOS 系统图标。

### 2.2 设计方向探索（三种草图方向）

| 方向 | 名称 | 核心图形 | 优劣势 |
|------|------|----------|--------|
| 方向 1 | **RC 字形设计** | 品牌首字母 "R" 构成主图形，粗圆线条，右侧三个对话圆点 | ✅ 品牌强、简洁、小尺寸清晰 / ❌ 需确保 R 字辨识度 |
| 方向 2 | **对话气泡 + 面具** | 白色气泡主形体，内部半透明面孔 | ✅ 语义完整 / ❌ 小尺寸面具无法辨认 |
| 方向 3 | **AI 星火 + 气泡** | 中心对称星形，外围气泡轮廓 | ✅ 科技感 / ❌ 与 ChatGPT 太相似，缺角色属性 |

**选定方向**：方向 1 —— RC 字形设计。品牌首字母具有唯一性，粗圆线条在大尺寸下有层次、小尺寸下不丢失识别。

## 三、草图设计

### 3.1 构思过程

1. 提取品牌首字母 **R**（RoleChat 的核心标识）
2. **R 自身隐喻为对话气泡**：竖线下端改为气泡尖角（像聊天气泡的三角尾巴），弧线头部形成气泡主体
3. 三圆点放在 R 弧线内部的负空间——像是"气泡内的输入中指示器"
4. R 的斜腿保持不变，保持字母辨识度
5. 风格：粗线条 + 圆端 + 投影 → 现代友好、不冷感

### 3.2 草图示意（文字描述）

```
┌──────────────────┐
│  ██████████████  │  ← 蓝色渐变背景
│  ██████████████  │
│      │▔▔╲        │
│      │ ● ╲       │  ← R 竖线底部有三角尖角（气泡尾巴）
│      │ ● ╱       │     弧线内三个圆点（气泡内的"输入中"）
│      │▁▁╱ ●     │
│      │╲          │     斜腿正常（保持R的识别）

```

## 四、低保真设计

### 4.1 矢量骨架

在 1024×1024 画布上完成 SVG 矢量稿：

| 元素 | 形状 | 尺寸/位置 |
|------|------|-----------|
| 背景 | 圆角矩形 rx=225 | 0,0 → 1024,1024 |
| R 竖线 | 直线 + 尖角 | x=-260, 竖线 -320→40, 尖角→-360,150 |
| R 头部 | 贝塞尔弧线 stroke=100 | 从 (-160,-320) 弧到 (10,100) |
| R 斜腿 | 直线 stroke=100 | (10,100) → (280,260) |
| 对话点 | 3 个圆 r=22 | 圆心 (-60,-180/-90/0) |

### 4.2 色彩方案（低保真阶段仅用单色）

| 色值 | 用途 |
|------|------|
| `#1557B0` → `#2B7DE1` | 背景渐变 |
| `#FFFFFF` | 前景图形 |
| `#0A3060` (30%) | 投影 |

## 五、高保真设计（视觉稿）

### 5.1 最终效果

在低保真骨架基础上添加：

- **背景**：45° 对角线渐变（深蓝 #1557B0 → 亮蓝 #2B7DE1），增强空间感
- **前景投影**：`feDropShadow`，dy=6, blur=8，深蓝 30% 透明度，增加图形浮起感
- **R 字形**：粗线条（100px 宽），圆端 + 圆角连接，视觉柔和
- **对话圆点**：三个竖向排列，透明度递减（0.9 → 0.55 → 0.25），模拟"对方正在输入"动画的静态快照
- **整体构图**：图形重心在画布上方 1/3 处（translate Y=500），符合 iOS 图标视觉重心规范

### 5.2 设计亮点

- **R 即气泡**：R 的竖线底部延伸为三角形尖角——视觉上就是一个对话气泡，字母与功能隐喻合二为一
- **内部负空间**：三圆点置于 R 弧线的白色负空间中，模拟气泡内的"对方正在输入"三点跳动
- **简约克制**：仅 5 个图形元素，小尺寸下 R 气泡轮廓 + 斜腿清晰可辨

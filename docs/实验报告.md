## 一、设计需求

### 1.1 项目背景

RoleChat 是一款通用型 AI 角色扮演对话平台。用户可设定世界观与角色，进入沉浸式对话。平台需一套功能图标承载所有交互入口，替代文字辅助导航。

## 二、设计定位

### 2.1 风格方向

采用 **极简线性风格（Linear Style）**。图标以一致的描边线条构成，必要时部分元素可填充以增强语义。风格参考 iOS 系统图标与 IconPark 线性图标集。

### 2.2 产品调性

- 目标平台：移动端优先 Web App（适配 480px 宽度的手机视口）
- 氛围定位：现代、轻量、沉浸式
- 色调体系：主蓝色 `#1A73E8`，背景浅灰 `#F5F5F7`
- 图标在界面中的角色：承载所有功能入口，替代或辅助文字标签

### 2.3 设计规范

| 规范项 | 参数 |
|--------|------|
| SVG 画布 | `viewBox="0 0 48 48"`（IconPark 导出默认画布） |
| 显示尺寸 | 24 × 24 px（CSS `--icon-size`）；小图标 14/18 px |
| 描边粗细 | `stroke-width="4"`（48 px 画布下为 4 px，IconPark 预设值） |
| 描边样式 | `stroke-linecap="round"`，`stroke-linejoin="round"` |
| 视觉平衡 | 图标重心居中，正负形比例均匀 |
| 默认态 | 描边色 `#8E8E93`，无填充 |
| 活跃态-导航 | 描边色切换至 `#1A73E8`，不填充 |
| 活跃态-赞 | 填充 `#1A73E8`，无描边（`.liked` 类触发） |
| 活跃态-踩 | 填充 `#FF3B30`，无描边（`.disliked` 类触发） |
| 禁用态 | 描边色 `#D1D1D6`，整体 `opacity: 0.4` |

## 三、草图与素材收集

### 3.1 参考来源

图标原型均选自 **IconPark（iconpark.oceanengine.com）**——字节跳动开源图标库，提供超过 2000 枚风格统一的线性/面性图标，支持在线调整描边粗细、圆角、尺寸后导出 SVG。

### 3.2 选取原则

在 IconPark 中按以下原则筛选每个功能的候选图标：

1. **语义准确**：图标隐喻直观，无歧义（如搜索选放大镜，发送选纸飞机）
2. **风格统一**：统一选择「线性」分类下的「描边」风格，确保线条粗细、端点样式一致
3. **正负形平衡**：筛选时对比多个候选，选取视觉重心居中、不过密不过疏的版本
4. **可识别性**：在小尺寸（24 px）下仍能清晰辨认

## 四、图标数字化设计与实现

### 4.1 工具与流程

1. 在 **IconPark** 官网（iconpark.oceanengine.com）搜索每个功能对应的关键词
2. 在线调整参数：描边粗细 4（IconPark 默认值，对应 48 px 画布下 4 px）、端点圆头、连接圆角
3. 导出 SVG 格式，命名遵循 `ic-{name}.svg` 规范存储至 `assets/icons/`
4. 对 SVG 源码进行统一处理：`stroke="#333"` → `stroke="currentColor"`，移除 XML 声明与固定宽高，保留 `viewBox="0 0 48 48"`
5. 将所有 SVG 内联嵌入 `index.html`，通过 CSS `color` 属性控制三态切换

### 4.2 颜色方案

| 状态 | CSS 变量 | 色值 | 触发方式 |
|------|----------|------|----------|
| 默认态 | `--color-icon-default` | `#8E8E93` | 所有 `.icon` 默认 |
| 活跃态-描边 | `--color-icon-active` | `#1A73E8` | `.nav-item.active .icon`、`.send-btn .icon` |
| 活跃态-赞填充 | `--color-primary` | `#1A73E8` | `.msg-actions button.liked .icon path` |
| 活跃态-踩填充 | `--color-danger` | `#FF3B30` | `.msg-actions button.disliked .icon path` |
| 禁用态 | `--color-icon-disabled` | `#D1D1D6` | `.icon-btn:disabled .icon` + `opacity: 0.4` |

### 4.3 三态切换机制

所有 SVG 路径统一使用 `stroke="currentColor"`，颜色由 CSS `color` 属性控制。**不依赖多套图片资源**：

```
SVG inline:  <path stroke="currentColor" stroke-width="4" .../>
```

| 状态 | CSS 规则 | 效果 |
|------|----------|------|
| 默认态 | `.icon { color: #8E8E93 }` | 灰色描边 |
| 导航活跃 | `.nav-item.active .nav-icon { color: #1A73E8 }` | 蓝色描边，不填充 |
| 赞激发 | `button.liked .icon path { fill: #1A73E8; stroke: none }` | 蓝色填充拇指 |
| 踩激发 | `button.disliked .icon path { fill: #FF3B30; stroke: none }` | 红色填充拇指 |
| 禁用态 | `.icon-btn:disabled .icon { color: #D1D1D6; opacity: 0.4 }` | 浅灰半透明 |

赞和踩互斥：点击赞自动清除踩状态，反之亦然。再点已选中按钮取消选中。

### 4.4 图标应用位置

所有 19 个图标均内联嵌入 HTML，分布在：

- **底部导航栏**（4 个）：对话、角色、发现、我的
- **顶部操作栏**（3 个）：新建（+号）、返回、角色切换
- **搜索栏**（1 个）：搜索
- **输入发送区**（1 个）：发送
- **消息气泡操作**（4 个）：刷新、复制、点赞、踩
- **角色创建表单**（4 个）：世界观、角色设定、用户身份、保存
- **对话/角色卡片操作**（2 个）：删除、收藏

## 六、设计总结与自评

### 6.1 规范性（权重 30%）

所有图标统一 `viewBox="0 0 48 48"` 画布、`stroke-width="4"` 描边、`stroke-linecap/join="round"` 圆角端点。颜色通过 CSS 变量集中管理，确保各状态色值严格一致。符合移动端 Web App 的图标尺寸与颜色规范。

### 6.2 视觉一致性（权重 25%）

图标全部选自 IconPark 同一风格分类（线性/描边），线条粗细、端点样式、圆角处理均一致。正负形比例经对比筛选后保持视觉面积相近。

### 6.3 可识别性（权重 20%）

选取的图标隐喻均为行业通用符号（放大镜 = 搜索、纸飞机 = 发送、五角星 = 收藏），预期可识别性通过率 ≥ 80%。实际测试待完成。

### 6.4 创新性（权重 15%）

在 CSS 层面创新实现单 SVG 三态切换（`currentColor` + 类名控制），避免维护多套图标资源。部分图标（如角色切换、世界观）为场景定制组合。

### 6.5 文档完整性（权重 10%）

本报告覆盖设计需求→定位→素材选取→数字化实现→输出的完整流程，所有图标均有使用场景和颜色标注记录。
